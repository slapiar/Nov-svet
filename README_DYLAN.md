# English orientation for Dylan

## Zephyr may already contain the first working layer of something much larger.

This repository is mainly written in Slovak, because it is the working space of the New World concept.

For Dylan, the important English entry points are:

1. [`INVITATION_FOR_DYLAN.md`](./INVITATION_FOR_DYLAN.md)  
   A short human invitation explaining why Zephyr matters here.

2. [`ZEPHYR_EN.md`](./ZEPHYR_EN.md)  
   The main English document mapping Zephyr Project Manager, Zephyr Pro and the mobile app to the first project-management layer of the New World system.

3. [`EMAIL_TO_DYLAN.md`](./EMAIL_TO_DYLAN.md)  
   A short email draft whose only purpose is to invite Dylan into the repository.

4. [`README_EN.md`](./EMAIL_TO_DYLAN.md)  
   The main English translation of Slovak README.md   

The core idea is simple:

```text
goal → promise → commitment → action → evidence → usefulness → authority → sustainable relationship
```

Zephyr already has projects, tasks, custom fields, files, comments, reports, frontend access and a mobile app.

That means it can already test goals, commitments, evidence and real-world progress.

The mobile app is especially important because it brings Zephyr closer to real action in the field — where photos, updates, comments and evidence can be captured as the work actually happens.

Zephyr is not presented here as the final New World system.

It is presented as the first practical workshop where this logic can be tested before building anything new from scratch.
